# BidOps 审核减负运行手册

## 目标

用“质量评分 + 异常优先复核 + 低风险批量确认”减少人工逐字段核对，但不绕过人工审核。AI 和规则结果仍只进入 Staging，Formal 入库必须由有权限的审核人员触发。

## 低风险批量确认

适用条件：

- 审核任务状态为待审核或审核中。
- `RiskLevel = Low`。
- `HighRiskIssueCount = 0`。
- 推荐动作为“可批量确认候选”。
- 任务未入库、未忽略、未合并。

操作：

1. 在待审核池筛选低风险任务。
2. 抽查同来源、同公告类型的样本。
3. 勾选任务并点击“低风险批量确认”。
4. 查看逐项结果；失败项保留在待审核池继续人工复核。

注意：

- 批量确认仍逐条走单任务审核逻辑，并写入纠错样本 `BulkApprove`。
- 高风险或状态变化的任务会被服务端拒绝，不会被批量通过。

## 高风险金额异常处理

优先复核以下 issue：

- `AmbiguousAmountUnit`：原始表头或单元格明确是万元/万，但暂存金额疑似未乘 10000。
- `RateOrDiscountInAmountColumn`：百分比、折扣率、费率、税率或得分进入金额字段。

处理方式：

1. 打开审核详情页的“异常复核”区域。
2. 对照原始公告、附件文本或原始行 JSON。
3. 若表头含“万元/万”，金额应按元存储。
4. 若原始值是百分比、折扣率或费率，不应写入金额字段。
5. 修正后可提交 DeepSeek 提示词重解析或人工编辑结果明细。

## 批量 DeepSeek 提示词重解析

适用场景：

- 同一批公告有相同表头别名或金额单位错误。
- 同一来源的候选/中标公告厂家或排名识别不稳定。
- 采购明细表头变化，需要一次性提示解析规则。

操作：

1. 在待审核池勾选任务。
2. 点击“批量提示词重解析”。
3. 输入明确、可复用的提示词，例如：“附件采购一览表中含万元的金额字段统一换算为元；最高限价（%）为费率，不写入金额字段。”
4. 提交后系统按公告类型入队：
   - 采购公告走结构化重解析。
   - 中标/候选公告走结果厂家重解析。

提示词会记录为纠错样本 `ReparsePrompt`，用于后续规则和 Prompt 优化。

## 历史质量回填

用途：

- 为旧的待审核任务补齐质量分、风险等级和异常项。
- 验证新规则对历史任务的影响。

操作：

1. 在待审核池点击“回填质量”入队最近 100 个待审核任务。
2. 也可通过 API 提交 `dryRun = true` 做演练。
3. 默认 `pauseSourceAware = true`，已暂停来源会跳过。
4. 回填只更新 ReviewTask 质量摘要和 `bidops_review_quality_issue`，不修改 Formal 数据。

## 纠错样本分析

入口：待审核池点击“质量分析”。

重点查看：

- 高频错误字段。
- 高频原始表头。
- 金额单位线索。
- 资质/业绩/人员要求线索。
- 最近人工编辑、批量确认、重解析提示词样本。

使用建议：

- 高频表头可转成解析别名字典或规则测试样例。
- 高频金额问题优先补解析单元测试。
- 高频提示词可沉淀为默认系统提示词，但需要先通过样例回归验证。

## 效率指标

质量分析页展示：

- 今日新增待审。
- 待审总数。
- 低/中/高风险数量。
- 低风险占比。
- 今日批量确认数量。
- 人工平均处理时长。
- 今日重解析提示词数量。

指标用于排班和规则改进，不用于自动通过或替代审核责任。
